AGUALUZ

Plataforma SaaS para la centralización operativa y gestión de distribución de agua potable.

CONTEXTO

Agualuz es una empresa de distribución de agua que gestionaba sus operaciones mediante llamadas telefónicas, WhatsApp y registros manuales. A medida que aumentaba el volumen de pedidos, coordinar clientes, operadores y conductores se volvía cada vez más complejo, generando errores operativos y dificultades para escalar el servicio.

DESAFÍO

Diseñar una plataforma capaz de centralizar pedidos, clientes, rutas y conductores dentro de un único sistema.

Además del desafío operativo, el proyecto presentó requisitos cambiantes durante gran parte del proceso, por lo que fue necesario estructurar flujos flexibles que mantuvieran una experiencia consistente para los usuarios a medida que evolucionaban las necesidades del negocio.

HALLAZGOS PRINCIPALES

Pedidos sin trazabilidad

Los pedidos podían perderse, eliminarse o quedar incompletos durante la gestión manual.

Definiciones operativas ambiguas

Conceptos fundamentales como las rutas no estaban completamente definidos entre los distintos stakeholders.

Comunicación fragmentada

Operadores y conductores dependían de múltiples canales para coordinar entregas y resolver incidencias.

Escalabilidad limitada

La operación dependía en gran medida de procesos manuales difíciles de sostener a medida que aumentaba el volumen de trabajo.

DECISIONES UX

Sistema de estados para pedidos

Diseñé una estructura basada en pedidos activos, archivados y eliminados para mejorar la trazabilidad y evitar pérdidas de información dentro de la operación.

Gestión flexible de rutas

Se definió un modelo que permitía asignaciones manuales inmediatas mientras preparaba el sistema para futuras automatizaciones.

Separación de información operativa

Se reorganizó la información de clientes y pedidos para reducir redundancias y facilitar la gestión diaria de los operadores.

Comunicación integrada

Se incorporó un flujo de seguimiento entre operadores y conductores mediante estados y notas compartidas dentro del sistema.

ARQUITECTURA DE INFORMACIÓN

Antes de diseñar las interfaces, estructuré la arquitectura del sistema para comprender la relación entre pedidos, rutas, conductores y administración.

Este mapa permitió organizar módulos, permisos y dependencias, asegurando coherencia entre la plataforma administrativa y la aplicación para conductores.

SOLUCIÓN

La propuesta se materializó en una plataforma SaaS orientada a operadores y administradores, acompañada por una aplicación móvil para conductores.

El sistema centraliza la gestión de pedidos, clientes, rutas y recursos operativos dentro de un flujo unificado que permite monitorear entregas, coordinar conductores y mantener la trazabilidad de cada pedido.
